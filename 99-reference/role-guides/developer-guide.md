# Developer Guide — AI-DLC

## Role Objective

Ensure AI-generated code is correct, secure, maintainable, and production-ready.

Developers are reviewers and refiners, not typists.

---

## Responsibilities by Phase

### Construction Phase

**Accountabilities**

* Review AI-generated code
* Refine implementations
* Enforce coding standards

**Key Activities**

* Participate in functional design reviews
* Review generated unit tests
* Optimize performance and security

**Artifacts Owned / Co-Owned**

* Source code
* Unit tests
* Build scripts

**Gate Involvement**

* Gate 5 — Code Generation Approval (support)
* Gate 6 — Build & Test Validation

---

## Operating Rules

* No manual code before plan approval
* All changes via Pull Requests
* No untested code merged

---

## What Developers Do Not Do

* Override phase gates
* Generate code without approval
* Treat AI output as final by default
